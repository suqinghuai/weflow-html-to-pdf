# Weflow 聊天记录 HTML 转 PDF 插件

## 插件介绍

这是一个专门为 **WEFLOW 4.3.0** 版本微信聊天记录导出软件设计的插件，用于将导出的 HTML 格式聊天记录转换为 PDF 格式，方便打印和取证使用。

## 功能特点

- 📁 **自动识别并处理同级目录下的所有 HTML 文件**
- 🔪 智能拆分大文件，避免转换失败
- 🔧 自动修复 HTML 打印格式问题
- 📄 批量转换为 A4 格式 PDF 文件
- 🧹 自动清理临时文件，保持目录整洁
- 🎉 友好的终端界面，操作简单直观
- ⏱️ 智能超时重试机制，转换更稳定可靠
- 🌐 增强的浏览器检测功能，支持更多浏览器版本
- 🔍 手动指定浏览器路径，解决自动识别失败问题
- 🐛 修复 Playwright 多线程兼容性问题，提升稳定性
- 🔄 **新增批量处理模式：支持处理多个文件夹，统一设置消息数量**
- 🛡️ **Windows 10 系统优化：增强批量处理稳定性，避免卡顿问题**
- 🔧 **JSON 解析增强**：自动清理无效控制字符，提升数据解析稳定性

## 系统要求

- Windows 10/11 操作系统
- 已安装 Microsoft Edge、Google Chrome 或 Chromium 浏览器之一
- WEFLOW 4.3.0 版本导出的 HTML 格式聊天记录

## 如何使用

### 步骤 1：准备工作

1. 使用 **WEFLOW 4.3.0** 导出微信聊天记录为 HTML 格式
2. 找到导出的 HTML 文件
3. 将 `HTML转PDF插件.exe` 文件复制到这个文件夹中
4. 双击运行 `HTML转PDF插件.exe` 

### 步骤 2：运行插件

1. 双击运行 `HTML转PDF插件.exe` 文件
2. 选择处理模式：
   - **模式1：处理当前目录** - 直接识别当前目录下的HTML文件
   - **模式2：批量处理多个目录** - 手动填写总目录路径，自动识别子目录
3. 根据选择的模式进行后续操作：
   - 模式1：输入每个文件包含的消息数量（默认 4000 条）
   - 模式2：输入总目录路径，然后统一设置消息数量（所有子文件夹使用相同设置）

### 步骤 3：等待转换完成

插件会自动执行以下操作：
1. 🔪 拆分 HTML 文件（如果文件过大）
2. 🔧 修复 HTML 打印格式
3. 📄 转换为 PDF 文件
4. 🧹 清理临时文件

转换完成后，会在每个处理目录中生成一个 `PDF输出` 文件夹，所有 PDF 文件都会保存在对应的文件夹中。

## 批量处理模式使用说明

### 适用场景
当您有多个聊天记录文件夹需要批量处理时，可以使用批量处理模式：

```
C:\聊天记录\
├── 张三聊天记录\
│   ├── 聊天记录1.html
│   └── 聊天记录2.html
├── 李四聊天记录\
│   └── 聊天记录3.html
└── 王五聊天记录\
    └── 聊天记录4.html
```

### 操作步骤
1. **选择模式2（批量处理多个目录）**
2. **输入总目录路径**（如：`C:\聊天记录`）
3. **统一设置消息数量**（只需输入一次，所有子文件夹使用相同设置）
4. **确认开始批量处理**

### 批量处理优势
- ✅ **一次性设置**：所有子文件夹使用相同的消息数量，避免重复输入
- ✅ **自动识别**：程序自动识别总目录下的所有子文件夹
- ✅ **独立输出**：每个子文件夹独立生成PDF输出文件夹
- ✅ **进度显示**：实时显示处理进度和结果统计

## 常见问题

### 问题：转换失败，提示没有找到可用浏览器

**解决方案**：
1. 确保已安装 Microsoft Edge、Google Chrome 或 Chromium 浏览器
2. 如果自动检测失败，程序会提示你手动输入浏览器路径
3. 常见浏览器路径示例：
   - Microsoft Edge: `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`
   - Google Chrome: `C:\Program Files\Google\Chrome\Application\chrome.exe`
4. 你可以直接拖拽浏览器 exe 文件到命令行窗口来自动填充路径
5. 如果仍然失败，可以尝试运行以下命令安装浏览器：
   ```
   playwright install msedge
   # 或
   playwright install chrome
   # 或
   playwright install chromium
   ```

### 问题：生成的 PDF 文件不完整或格式不正确

**解决方案**：
1. 尝试减小每个文件包含的消息数量（建议 2000-3000 条）
2. 确保浏览器可以正常打开和渲染 HTML 文件

### 问题：转换过程中超时或卡住

**解决方案**：
1. 程序会自动进行最多 3 次重试，每次超时时间为 10 分钟
2. 如果 3 次重试都失败，程序会提示你选择：
   - 重试：重新尝试转换当前文件
   - 跳过：跳过当前文件，继续处理下一个文件
   - 退出：退出整个程序
3. 建议减小每个文件包含的消息数量（建议 2000-3000 条）以避免超时

### 问题：程序运行后没有反应

**解决方案**：
1. 检查是否有其他程序占用了浏览器资源
2. 确保当前目录下有 HTML 文件
3. 尝试以管理员身份运行程序

### 问题：批量处理模式下，Windows 10 系统转换完第一个文件夹后卡住

**解决方案**：
1. **已修复**：程序已增强资源释放机制，确保每个文件夹处理完成后正确清理浏览器上下文
2. **已优化**：添加系统资源强制回收和适当延迟，避免浏览器进程残留
3. **建议**：如果仍遇到问题，可以尝试重启程序或重启计算机释放系统资源

### 问题：JSON解析错误，提示"Invalid control character"

**解决方案**：
1. **已修复**：程序已增强JSON解析功能，自动清理数据中的无效控制字符
2. **说明**：这通常是由于聊天记录中包含不可见的特殊字符导致的
3. **建议**：如果仍遇到问题，请检查HTML文件是否损坏或包含异常字符
